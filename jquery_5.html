제이쿼리 xml

xml은 웹을 통해 문서를 표현하고 전달하기 위해 설계된 표준 마크업 언어입니다 xml은 html5와 다르게 문서 내용에 의미와 구조를 명세하는 마크업을 자유롭게 추가할수 있습니다.

웹은 단순 정보를 제공 수단에서 점차 정보 서비스 토대로서 중요성이 커지고 있습니다 웹을 통한 정보 교환이 빈번해지고 다양해짐에 따라 웹 정보의 구조와 의미를 보다 명확히 처리 할수있는 기술이 필요하다 xml은 정보 표현이 자유롭고 또 다양한 환경에서도 인식이 가능하다는 장점이 있습니다 웹 정보 교환의 핵심 기술로 인식되고 있습니다.

xml 예제

xml의 구조선언문은 doctpye productinfo system"product 이며 문서 선언은 xml version="#" encoding="UTF-8" 입니다 html5 와 비슷합니다 xml은 반드시 최상위 엘리먼트를 하나만 갖는 특징이 있고 엘리먼트를 감싸는 역활을 한다 이는 순차적으로 진행하는 특징이 있으며 태그들의 중첩이 불가능하며 순차적으로 작동해야 한다는 조건부가 붙습니다 이 부분을 망각하고 제작한다면 큰 차질이 생길것입니다. 개인적으로 xml dom의 트리 구조의 이해를 하신다면 작성원리를 쉽게 이해하실수있습니다
